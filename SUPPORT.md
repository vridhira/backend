# Support

## Before Asking for Help

1. Check [README.md](./README.md).
2. Review [CONTRIBUTING.md](./CONTRIBUTING.md).
3. Confirm your issue is not already reported.

## Support Channels

1. Bug reports and feature requests: GitHub Issues (use templates).
2. Security vulnerabilities: private email only (see [SECURITY.md](./SECURITY.md)).
3. Licensing/commercial use inquiries: hk8913114@gmail.com.

## What to Include

For technical issues, include:

1. Environment details (OS, Node, package manager).
2. Exact error messages.
3. Reproduction steps.
4. Relevant logs and screenshots.

## Response Expectations

1. Community support is best-effort.
2. Critical security reports are prioritized.
3. Commercial support terms can be handled separately via email.
