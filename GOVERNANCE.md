# Governance

## Project Ownership

This repository is maintained by:

- Owner: @Newbie-Himanshu

## Decision Model

1. Maintainer-led decisions for architecture, security, and licensing.
2. PR feedback is considered, but merge authority remains with maintainers.
3. Security and licensing files are protected via CODEOWNERS and branch rules.

## Contribution Flow

1. Open issue (or use existing one).
2. Open focused PR from a branch based on `main`.
3. Pass required checks.
4. Obtain required reviews.
5. Merge through protected branch workflow.

## Protected Areas

Changes to these areas require strict review:

1. Licensing and legal documents.
2. Security-sensitive modules (auth, payments, webhooks).
3. Branch protection/workflow policies.

## Enforcement Principles

1. Security over speed.
2. Clear traceability in commits and PRs.
3. Fail-closed defaults in sensitive paths.
4. Respect for project licensing requirements.
